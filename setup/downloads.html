Setup and Downloads

Install the minimum tools for the first lessons. Docker is optional for Lesson 1, but recommended for later Postgres, Redis, and stream labs.

Minimum setup for Lesson 1

required

Python 3

Use Python 3.10+ for the DuckDB lab. Check your version:

python --version
# or
python3 --version

Official download: python.org/downloads

required

DuckDB Python package

Install inside a virtual environment:

python -m venv .venv
source .venv/bin/activate   # macOS/Linux
# .venv\Scripts\activate    # Windows PowerShell
pip install duckdb pandas pyarrow

Package page: PyPI duckdb

Recommended setup for later lessons

recommended

Docker Desktop

Docker lets us run Postgres, Redis, and Redpanda/Kafka-like services without installing each one directly on your machine.

Official install guide: Docker Get Docker

docker --version
docker compose version
later labs

Postgres client

Docker can run the server, but a local SQL client is useful. You can use psql, DBeaver, DataGrip, or VS Code database extensions.

psql --version

Local services with Docker Compose

The repo includes labs/docker-compose.yml. Later lessons will use it to start local systems.

cd labs
docker compose up -d postgres redis redpanda
docker compose ps

docker compose down

Use Docker carefully: containers are useful for learning, but they still run code on your machine. Use official images, avoid mounting sensitive folders, and stop containers you are not using.

How submissions work

Each lesson has answer boxes. Open the lesson locally, type your answers, then use the buttons at the bottom:

  • Save Draft stores answers in your browser local storage.
  • Copy Submission copies a Markdown version to your clipboard.
  • Download Submission downloads a .md file you can commit or paste into chat.

Browser local storage is not Git. Download or copy your answers before switching browsers or clearing site data.
