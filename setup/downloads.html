Setup and Downloads

Install only what the current lesson needs. Lesson 1 uses Python and DuckDB; Lessons 2 and 3 use PostgreSQL through Docker.

Python setup

required

Python 3

Use Python 3.10+ for the DuckDB lab and hot-partition simulator.

python --version
# or
python3 --version

Official download: python.org/downloads

Lesson 1

DuckDB packages

Install inside a virtual environment:

python -m venv .venv
source .venv/bin/activate   # macOS/Linux
# .venv\Scripts\activate    # Windows PowerShell
pip install duckdb pandas pyarrow

The Lesson 2 hot-key simulator uses only the Python standard library.

Docker and PostgreSQL for Lessons 2 and 3

Lessons 2–3

Docker Desktop

Docker runs PostgreSQL in a reproducible local container.

Official install guide: Docker Get Docker

docker --version
docker compose version
optional client

Local SQL client

The container already includes psql. DBeaver, DataGrip, or a VS Code extension is optional.

docker exec -it ds_course_postgres \
  psql -U app -d system_design

Start PostgreSQL

cd labs
docker compose up -d postgres
docker compose ps
cd ..

Lesson 2 setup

docker exec -i ds_course_postgres \
  psql -U app -d system_design \
  < labs/partitioning-postgres/01_setup.sql

Lesson 3 setup

docker exec -i ds_course_postgres \
  psql -U app -d system_design \
  < labs/transactions-postgres/01_setup.sql

Lesson 3 requires two interactive shells so you can create real concurrent transactions:

# Run in two terminal windows
docker exec -it ds_course_postgres \
  psql -U app -d system_design

Stop services when finished:

cd labs
docker compose down

Use Docker carefully: use trusted images, avoid mounting sensitive folders, and stop services you are not using.
