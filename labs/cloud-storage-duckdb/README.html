Lab: DuckDB + Parquet

Simulate a tiny cloud analytics workflow locally.

Goal

Generate partitioned Parquet files, query them with DuckDB, and reason about scans, pruning, and storage layout.

Setup

python -m venv .venv
source .venv/bin/activate      # macOS/Linux
# .venv\Scripts\activate       # Windows PowerShell
pip install duckdb pandas pyarrow

Run

python generate_data.py
python run_queries.py

Record these observations

  1. Which query is slowest?
  2. Which query appears to read less?
  3. Does filtering on day help?
  4. How would results change if the data were one huge CSV?
  5. What does this teach you about warehouse storage design?
